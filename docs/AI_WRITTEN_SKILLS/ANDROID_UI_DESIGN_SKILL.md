# ANDROID_UI_DESIGN_SKILL - 高质量移动端 UI 设计（项目工程约束）

## 来源与用途

本 Skill 是项目内自写通用 AI 参考规范，不是外部下载 Skill。仅保留本项目特有的工程约束，通用 UI 设计知识见外部 Skill `ui-android`。

## 本项目工程约束

- 做出有辨识度、可长期使用的本地媒体库 UI，避免普通模板感
- 先保证信息层级，再做装饰效果；媒体内容是主角，按钮和卡片不能喧宾夺主
- 多主题要有明显差异，不只是换主色
- 毛玻璃、胶囊、渐变、贴纸要统一，不要每页一套
- 深色主题必须考虑夜间可读性；浅色主题必须避免低对比灰字
- 禁止用随机渐变堆满页面
- 禁止在同一页面混用过多字体大小和颜色
- 禁止只做"白底列表 + 默认按钮"的普通工具 UI
- 禁止为了美观牺牲文件名、作者、统计等核心信息可读性
